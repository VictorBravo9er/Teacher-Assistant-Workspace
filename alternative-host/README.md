# Alternative Hosting Strategy

This strategy packages the compiled frontend application and serves it directly from the FastAPI backend. This is particularly useful for unified deployments where you want to run a single service (the Python backend) that handles both the API and serves the frontend assets, reducing the complexity of the deployment infrastructure.

## Workflow

1. **Build the Frontend**: The frontend (typically a React/Vue app) is built into static files (HTML, CSS, JS) inside a `dist/` directory.
2. **Move to Workdir**: The `dist/` directory is moved into the backend's working directory (`workdir/dist`).
3. **Serve with FastAPI**: The FastAPI application is configured with:
   - A static files mount for the `/assets` route (or any other static directories generated by your build tool).
   - A catch-all route (`/{full_path:path}`) that serves the `index.html` file. This is crucial for single-page applications (SPAs) because it allows the client-side router (e.g., React Router) to handle routing for any paths that aren't specific API routes.

## Files

- `deploy.sh`: A shell script that demonstrates building the frontend and moving the output to the backend `workdir`.
- `main.py`: A demo FastAPI script demonstrating how to serve the static assets and the catch-all `index.html` route.
